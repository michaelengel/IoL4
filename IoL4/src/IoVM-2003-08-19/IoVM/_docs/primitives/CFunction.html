Summary

    A container for a pointer to a C function binding. They can only be defined from the C side. They act like blocks in that when placed in a slot, they are called when the slot is activated. The "for", "if" "while" and "clone" methods of the Lobby are examples of CFunctions. CFunctions are useful for implementing methods that require the speed of C or binding to a C library.

Methods


    type

    Returns the string "CFunction".


    id

    Returns a number containing a unique identifier for this CFunction. This is constant during the lifetime of the program, but may not be same between different runs of the same program.


    ==(aFunction)

    Returns self if aFunction is a CFunction and its function pointer is equal to that of the receiver. Otherwise returns Nil.
